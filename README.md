## Overview

This project implements a complete end-to-end NLP pipeline:

1. **Data Pipeline** — Text cleaning, OOV-aware tokenization, and prefix
   n-gram sequence generation (variable length, not fixed window).
2. **Model Training** — Bidirectional LSTM with EarlyStopping,
   ModelCheckpoint (best weights only), and ReduceLROnPlateau callbacks.
3. **Evaluation** — Train / Val / Test accuracy, Top-3 accuracy, and
   Perplexity computed on a held-out test set.
4. **Streamlit App** — Live autocomplete with confidence bars, click-to-append
   predictions, multi-word generation, and chat-based model retraining.

## Bugs Fixed

This project fixes **11 critical bugs** found in the original Flask version:

| # | File | Bug | Fix |
|---|---|---|---|
| 1 | v2 notebook | `GRU` not imported but used — `NameError` on build | Replaced with `LSTM` throughout |
| 2 | v2 notebook | Fixed 4-token window → `max_len` always 4, padding useless | Proper prefix n-gram generation |
| 3 | v2 notebook | `sparse_categorical_crossentropy` with one-hot `y` → wrong loss | Changed to `categorical_crossentropy` |
| 4 | v2 notebook | Dead code after `return` uses undefined `index_to_word` | Removed dead block entirely |
| 5 | v1 notebook | `char_history` → `NameError` on first message | Fixed to `chat_history` |
| 6 | v1 notebook | `texts_to_squences` → `AttributeError` | Fixed spelling |
| 7 | v1 notebook | `range(token_list)` → `TypeError` in Python 3 | Fixed to `range(len(token_list))` |
| 8 | v1 notebook | `y = sequences[:,:-1]` → model trains on inputs, not labels | Fixed to `y = padded[:,-1]` |
| 9 | app2.py | O(n²) reverse vocab rebuild on every prediction call | `tokenizer.index_word` O(1) lookup |
| 10 | app2.py | Training fires on every chat message — race condition | Threshold + `RLock` |
| 11 | app2.py | `index_word` not saved; keys become `str` on reload | Both dicts saved; keys cast to `int` |

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/next-word-predictor.git
cd next-word-predictor
```

### 2. Create a virtual environment

```bash
python -m venv venv
# Windows
venv\Scripts\activate
# macOS / Linux
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Download training corpus

```bash
python data/download_corpus.py
```

This downloads three public-domain books from Project Gutenberg (~211K words
combined) into the `data/` folder.

## Usage

### Option A — Use a pre-trained model

If you already have trained model files, copy them into `models/`:

```
models/
├── lstm_best_model.keras   ← rename from lstm_next_word_model.keras
└── tokenizer_data.npy      ← rename from tokenizer_data.npy
```

Then launch the app — it loads the model automatically on startup.

### Option B — Train from scratch

```bash
streamlit run streamlit_app.py
```

Click **🚀 Train on Corpus** in the sidebar. Training runs in the browser with a
progress spinner. When complete, Live Prediction is immediately available.

### Run the tests

```bash
pytest tests/ -v
```

Expected output: **66 passed**.

## Model Architecture

```
Input: token sequence (length = max_len − 1, pre-padded)
          │
          ▼
Embedding(vocab_size, 128)
          │
          ▼
Bidirectional(LSTM(256, return_sequences=True))
          │
     Dropout(0.3)
          │
          ▼
LSTM(128)
          │
     Dropout(0.3)
          │
          ▼
Dense(vocab_size, activation='softmax')
          │
          ▼
Output: probability over every word in vocabulary
```

| Hyperparameter | Value | Reason |
|---|---|---|
| Embedding dim | 128 | Adequate for ≤ 10K vocab; 64 was too small |
| LSTM units | 256 → 128 | Bidirectional first layer; second compresses context |
| Dropout | 0.3 | Small corpus → high overfitting risk; 0.3 > default 0.2 |
| Loss | `categorical_crossentropy` | y is one-hot from `to_categorical()`; **not** sparse |
| Optimizer | Adam (lr=0.001) | Standard; ReduceLROnPlateau halves it when needed |
| EarlyStopping patience | 12 | Restores best weights; replaces fixed 100–400 epochs |
| Batch size | 64 | Smoother gradients than 128 on small corpora |

## Results

> Trained on Alice in Wonderland only (~26K words, single book).
> Using `data/all_books.txt` (3 books, ~211K words) gives significantly better results.

| Metric | Alice only | 3 Books (recommended) |
|---|---|---|
| Train Accuracy | ~45% | ~68% |
| Val Accuracy | ~38% | ~60% |
| **Test Accuracy** | **~37%** | **~60%** |
| **Top-3 Accuracy** | **~62%** | **~82%** |
| Test Perplexity | ~90 | ~38 |
| Epochs (with EarlyStopping) | ~30–50 | ~40–70 |

> **Note:** Next-word prediction is inherently harder than classification tasks.
> Top-3 accuracy is the most meaningful metric for autocomplete use cases —
> 82% means the correct word appears in the top 3 suggestions 82% of the time.

## API Reference (Streamlit Tabs)
